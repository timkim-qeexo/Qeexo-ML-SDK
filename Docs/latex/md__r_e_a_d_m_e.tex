The SDK aims to read various types of sensor data, fill the sensor buffer, and deliver it to multiple registered applications. Each application processes the sensor data in an independent task. The project name is Qeexo-\/\+ML-\/\+SDK and it should be registered with a repository in Git.\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{System Hierarchy Structure}{System Hierarchy Structure}}\label{md__r_e_a_d_m_e_autotoc_md1}

\begin{DoxyEnumerate}
\item HAL Structure
\begin{DoxyEnumerate}
\item Overview
\begin{DoxyEnumerate}
\item All HAL codes should be implemented in C language.
\item The prefix "{}\+Qx"{} is used, and in case the module name is attached, for example, HAL source code related to OS will be Qx\+OS.\+c and Qx\+OS.\+h files.
\item Function names should also use the same prefix.
\item Create a folder named "{}\+Hal"{} and design the structure and files as outlined below.
\end{DoxyEnumerate}
\item Qx\+OS.\+h, Qx\+OS.\+c
\begin{DoxyEnumerate}
\item Provide OS-\/related APIs.
\item The files should include the following functions\+:
\begin{DoxyEnumerate}
\item BSP initialization function
\item Thread creation and termination, suspend, resume functions
\item Mutex creation, termination, lock, unlock functions
\item Mailbox related functions
\item Timer related functions
\end{DoxyEnumerate}
\end{DoxyEnumerate}
\item Qx\+Sensor\+Hal.\+c, Qx\+Sensor\+Hal.\+h
\begin{DoxyEnumerate}
\item A module that manages and registers sensors.
\item Each sensor should implement the following functions and register function pointers in a structure so that the upper framework can load and execute each function\+:
\begin{DoxyEnumerate}
\item init function
\item enable function with odr and fsr as parameters
\item disable function
\item read function
\item read\+\_\+fifo function
\item get\+\_\+odrs function, providing a list of supported ODRs
\item get\+\_\+fsrs function, providing a list of supported FSRs
\end{DoxyEnumerate}
\end{DoxyEnumerate}
\item Qx\+Transceiver\+Hal.\+c, Qx\+Transceiver\+Hal.\+h
\begin{DoxyEnumerate}
\item A module that registers and manages communication media such as USB, BLE, and Wi\+Fi, which should be implemented as sub-\/drivers and registered.
\item Each transceiver should be implemented with common functionality and connected through function pointers, managed by Qx\+Transceiver\+HAL, and callable by upper applications\+:
\begin{DoxyEnumerate}
\item Open function
\item write function
\item read function
\end{DoxyEnumerate}
\end{DoxyEnumerate}
\end{DoxyEnumerate}
